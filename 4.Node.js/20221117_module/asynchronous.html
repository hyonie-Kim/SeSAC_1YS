비동기(asynchronous)

  • 비동기 코드는, 코드의 순서와 다르게 실행됨
  • 비동기 처리 코드를 감싼 블록은 task queue 에 넣어짐
  • main thread가 동기 코드를 실행한 후에 제어권이 돌아왔을때
    event loop가 task queue에 넣어진 비동기 코드를 실행함

예시]

      
        setTimeout(()=> console.log("This is asynchronous..."),5000);
        console.log("This is synchronous...");

        for(let i=0; i < 1000000000; ++i) {
          console.log("I am blocking the main thread...")
        }

예시로 setTimeout이 5초가 아니라 0초 일때, 동작을 예측하자면 for문이 끝날 때까지 main thread가 반환되지않았기 때문에
setTimeout이 바로 0초뒤에 콜백이 호출되겠끔, task queue에 넣어진다고 하더라도 main thread에 여유가 없으므로 task queue는 비워질수가 없다.
그래서 콜백은 0초 보다도 지난 시간에 호출이 될것이다.main thread의 여유가 있을때, 실행이 된다.
